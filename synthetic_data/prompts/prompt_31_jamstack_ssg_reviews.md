## LLM Generation Prompt

Generate a complete, self-contained HTML page representing a review section from a JAMstack or statically-generated site (2018–2024). These pages are pre-rendered at build time using Gatsby, Next.js (static export), Hugo, Eleventy, or Astro. The HTML is clean, semantic, and fast — but has distinctive fingerprints: Gatsby's `gatsby-image` wrappers, Astro's `data-astro-cid` attributes, Hugo's template-generated class names, or Eleventy's bare HTML with utility CSS. Reviews are often sourced from a headless CMS (Contentful, Sanity, Airtable) and baked into the static HTML at build time. Each of the 100 pages must vary across ALL of the following dimensions.

### Framework Fingerprints to Vary (pick one per page)

**Gatsby:**
- `<div class="gatsby-image-wrapper">` around images with `style="position: relative; overflow: hidden;"`
- Low-quality image placeholder: `<img src="data:image/png;base64,..." style="position: absolute; top: 0; left: 0; width: 100%; height: 100%; object-fit: cover; object-position: center center; opacity: 1; transition: none 0s ease 0s;">`
- `gatsby-highlight` class on code blocks
- Build metadata in HTML comment: `<!-- Gatsby Build: 2022-03-15T10:22:31.000Z -->`
- GraphQL data embedded in `<script id="gatsby-script-*">` tags

**Next.js Static Export:**
- `<div id="__next">` root
- `<script id="__NEXT_DATA__" type="application/json">{"props":{"pageProps":{...}},"page":"/reviews/[product]","buildId":"abc123XYZ"}</script>`
- Next.js Image: `<img src="/_next/image?url=...&w=828&q=75" srcset="...">` with width/quality params
- `<link rel="preload" as="script" href="/_next/static/chunks/...js">`
- `data-nimg` attribute on Next.js `<Image>` components

**Astro:**
- `data-astro-cid-XXXXXXXX` attribute on component elements (hashed component ID)
- `<astro-island>` or `<astro-slot>` custom elements for partial hydration islands
- Astro's `<style>` tag with scoped hash: `<style>h2[data-astro-cid-abcd1234]{color:#333}</style>`
- Script modules: `<script type="module" src="/_astro/ReviewCard.XXXXXXXX.js"></script>` for interactive islands

**Hugo / Eleventy (no JS fingerprints):**
- Hugo: `<!-- Generated by Hugo 0.111.3 -->` comment in head; templated class names like `class="review-card review-card--5-star"`
- Eleventy: minimal output, utility CSS (Tailwind), no framework attributes
- Very clean semantic HTML: `<article class="review">`, `<header class="review-header">`, `<footer class="review-meta">`

### Structural Dimensions to Vary
- Number of reviews: 6–30
- Review sourcing comment: `<!-- Reviews loaded from Contentful CMS at build time -->` or `<!-- Sourced from Airtable base: appXXXXXXXXXX -->`
- Tailwind CSS class names on ~40% of pages (common in JAMstack era): `class="flex flex-col gap-4 p-6 bg-white rounded-xl shadow-sm border border-gray-100"`
- CSS custom properties (variables): `--rating-color: #F59E0B; --card-radius: 0.75rem;` in `:root`
- Dark mode support: `class="dark:bg-gray-800 dark:text-gray-100"` or `@media (prefers-color-scheme: dark)` in CSS
- Schema.org JSON-LD: always present (JAMstack sites often auto-generate from CMS data)
- ARIA: comprehensive — `role="list"`, `aria-label`, `aria-describedby` standard practice by this era
- Review pagination: static paged URLs (`/reviews/page/2/`) with `<link rel="next">` in head

### Content Dimensions to Vary
- DTC brands, SaaS products, indie maker tools, creator economy platforms (Gumroad, Patreon alternatives)
- Review length: short modern social-media-influenced ("10/10 no notes") to detailed structured
- CSS framework: Tailwind (40%), custom CSS (30%), CSS Modules output (20%), Bootstrap 5 (10%)
- Build year implied: vary 2019–2024 based on framework versions and class name patterns

### Output Format
Single complete HTML file. Include appropriate framework fingerprints in `<head>` (preload links, build IDs, etc.). Use realistic JAMstack build artifacts in the markup. Include a product/service header with aggregate score and the review list. Include `<script type="application/ld+json">` structured data.
