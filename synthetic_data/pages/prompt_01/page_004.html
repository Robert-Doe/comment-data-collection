Setting up Apache 1.3 on Slackware
kernel_panic_99  |  Posts: 634  |  Senior Member  |  Posted: 14 Aug 2003 09:05
I've been banging my head against this for two days. I compiled Apache 1.3.28 from source on Slackware 9.0 and it starts fine but I get a 403 Forbidden on my document root. Permissions look right, DocumentRoot is set to /var/www/htdocs, directory has 755. Error log just says [error] client denied by server configuration: /var/www/htdocs

Anyone seen this before? I'm sure it's something obvious that I'm missing.
sysadmin_bob  |  Posts: 2,211  |  Guru  |  Posted: 14 Aug 2003 09:34
Check your Directory block in httpd.conf. By default Apache 1.3 denies access to everything. You need something like:

<Directory "/var/www/htdocs">
  Options Indexes FollowSymLinks
  AllowOverride None
  Order allow,deny
  Allow from all
</Directory>

Without the Allow from all line it defaults to deny. Classic gotcha.
kernel_panic_99  |  Posts: 634  |  Posted: 14 Aug 2003 10:02
That was it! I feel like an idiot. I had the Directory block but it said Deny from all with no Allow line at all. Everything works now. Thanks sysadmin_bob, you're a lifesaver.
tux_forever  |  Posts: 89  |  Posted: 15 Aug 2003 14:22
Worth mentioning for anyone else reading this: if you're using .htaccess files make sure AllowOverride is set to at least AllowOverride AuthConfig or whatever directives you're using. With it set to None your .htaccess is completely ignored and that causes all sorts of mysterious 403s too.
Anonymous  |  Guest  |  Posted: 17 Aug 2003 08:45
Also check SELinux if you're running a kernel with it enabled. Even with correct Apache permissions SELinux can block access. getsebool httpd_enable_homedirs and related booleans are worth checking. Not relevant on Slackware 9 but good to know for the future.
