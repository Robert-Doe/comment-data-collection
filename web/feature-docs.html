Feature Docs

Use this page as a plain-language reference for the modeling vocabulary, the label meanings, the two model variants, and every curated feature that the first training system uses.

Core Terms

Comment List

The list-level container that holds repeated comment blocks, review blocks, reply blocks, or answer blocks. This is the first model target.

Comment Block

One repeated unit inside the list, usually containing author, timestamp, controls, and the user-authored text.

Comment Block Element

The text-bearing node or wrapper that holds the user’s actual message, opinion, review text, or reply text. It does not include usernames, timestamps, avatars, or reply buttons.

Label Meanings

comment_region

The candidate is the real list-level comment region for the page.

not_comment_region

The candidate is not the right list-level comment region, even if it looks structured.

uncertain

The candidate is ambiguous. The first model does not train on it; it stays available for later review.
